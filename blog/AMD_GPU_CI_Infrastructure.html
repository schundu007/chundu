01

Scope & ownership model

This reference describes the complete CI infrastructure for a GPU stack release pipeline where firmware, kernel driver, and the ROCm compute runtime must be validated together across a fleet of GPU products. It is deliberately high-ownership: one platform engineer designs the architecture, provisions everything as code with Terraform, secures every layer (encryption, signing, audit logging), and owns reliability targets — a >95% nightly pass-rate and a <24h mean-time-to-detect on regressions.

The AMD role is explicit that nobody else on the team is currently focused on CI — so this is about setting direction and shipping the automation everything else depends on, not maintaining someone else's system.

Three core responsibilities

Nightly integration CI

Firmware + driver + ROCm validated together across GPU families, promoting a Last-Known-Good baseline.

Self-hosted runners

GitHub Actions runners needing NFS, signing access, and network isolation — provisioned as code.

AWS-aligned foundations

Terraform IaC, least-privilege IAM, encryption everywhere, audit-ready trails.

Agentic throughput

AI coding agents as a force multiplier on high-volume automation — verified behind review.

What the platform role owns

  • Architecture decisions — GitHub Actions vs. alternatives, Terraform vs. CloudFormation, runner topology, cache strategy.
  • Infrastructure provisioning — VPC, subnets, security groups, IAM, EC2, EFS, KMS, Secrets Manager.
  • Runner provisioning — Actions registration, NFS mounts, signing access, ccache/DVC caches, auto-shutdown.
  • CI/CD workflows — pre-submit gates, nightly integration, release-candidate validation, LKG manifest promotion.
  • Security & compliance — HSM firmware signing, SBOM, vulnerability scanning, audit logging.
  • Hardware integration — AMD Instinct device management, flashing, testing, thermal monitoring.
  • Operational excellence — observability, runbooks, failure-mode handling, cost control.

Why self-hosted runners?

GitHub-hosted runners can't support firmware builds — and they can't touch AMD hardware. The build and validation environment needs:

  • NFS mounts for cross-compiler toolchains and shared build caches (ccache, DVC-managed MIOpen kernels).
  • Signing access via CloudHSM / Secrets Manager — the private key never leaves hardware.
  • Network isolation — no inbound internet, private subnets, egress via a single controlled NAT.
  • Real AMD Instinct silicon for hardware-in-the-loop, which lives in a bare-metal lab, not a cloud instance.
JD alignmentThis is exactly the "runner provisioning problem" the role calls out: standard cloud runners can't build firmware, so the work is provisioning self-hosted runners that handle NFS, signing pipelines, network permissions — and coordinating with IT on the real constraints.

This isn't a hypothetical. AMD's unified GPU-stack build system, ROCm/TheRock ("The HIP Environment and ROCm Kit"), is public. name: Firmware Pre-Submit Validation
on:
  pull_request:
    branches: [main]
    paths: ['firmware/**']

jobs:
  build-and-validate:
    runs-on: [self-hosted, linux, firmware-builder]

      - name: Load signing secrets from Secrets Manager
        run: |

            --secret-id /rocm-ci/signing-keys \

      - name: Restore ccache
        uses: actions/cache@v4

          path: .ccache
          key: ccache-fw-${{ runner.os }}-${{ github.sha }}
          restore-keys: ccache-fw-${{ runner.os }}-

      - name: Build firmware (all targets)
        run: |
          eval "$(./build_tools/setup_ccache.py)"
          cd firmware
          cmake -B build -GNinja -DFIRMWARE_TARGETS=gfx908,gfx90a,gfx942
          cmake --build build -j"$(nproc)"

      - name: Generate SBOM (CycloneDX)
        run: syft firmware/build -o cyclonedx-json > firmware-sbom.json

      - name: Compatibility matrix check
        run: |
          python3 scripts/validate_compat.py \
            --firmware-dir ./firmware/build \
            --manifest-url s3://rocm-manifests/lkg.yaml

      - name: Static security analysis

          kics scan -p firmware/ -o json > kics.json

name: Nightly Integration Build
on:
  schedule:
    - cron: '0 6 * * *'   # after upstream submodule syncs

jobs:
  build-all-components:
    runs-on: [self-hosted, linux, rocm-builder]

      - name: Fetch pinned sources

      - name: Build full stack (ccache)

            -DCMAKE_C_COMPILER_LAUNCHER=ccache \
            -DCMAKE_CXX_COMPILER_LAUNCHER=ccache .
          cmake --build build -j"$(nproc)"
          DIGEST=$(sha256sum build/dist/*.tar.xz | awk '{print $1}')
          echo "digest=$DIGEST" >> "$GITHUB_OUTPUT"

        run: aws s3 cp build/dist/ s3://rocm-artifacts/nightly-$(date +%Y%m%d)/ --recursive --sse aws:kms

    runs-on: [self-hosted, instinct]        # bare-metal AMD Instinct lab
    strategy:
      matrix:
        gpu: [MI300X, MI250X, MI210, MI100]
      max-parallel: 2
      fail-fast: false                       # one family's failure ≠ block all
    steps:
      - name: Flash firmware to GPU
        run: |
          python3 scripts/gpu_firmware_update.py \
            --gpu-model ${{ matrix.gpu }} \
            --firmware ./artifacts/firmware.bin --verify
      - name: Run GPU validation
        run: |
          python3 test_harness.py --gpu-model ${{ matrix.gpu }} \
            --output junit-${{ matrix.gpu }}.xml
      - uses: actions/upload-artifact@v4

    needs: [build-all-components, hardware-validation]
    if: success()
    runs-on: [self-hosted, linux, rocm-builder]
    steps:
      - uses: actions/checkout@v4
        with: { repository: org/rocm-manifest, ref: lkg-main }
      - name: Generate LKG manifest
        run: |
          python3 scripts/update_manifest.py \
            --firmware-digest ${{ needs.build-all-components.outputs.firmware-digest }}
      - name: Sign manifest with GPG
        run: |
          gpg --import /var/secrets/rocm-ci-gpg-key
          gpg --sign --detach-sign --armor manifest.yaml
      - name: Commit and push
        run: |
          git config user.name 'rocm-ci-automation'
          git add manifest.yaml manifest.yaml.asc
          git commit -m "Update LKG: ${{ needs.build-all-components.outputs.firmware-digest }}"
          git push origin lkg-main

DevOps security & code signing

Security is not optional. Firmware signing via CloudHSM (key never leaves hardware), SBOM generation, vulnerability scanning, and complete audit trails.

  1. Build firmware on an isolated runner (no internet, /tmp as ramfs).
  2. Compute a SHA-256 hash of the binary.
  3. Sign the hash via AWS CloudHSM (VPC endpoint — private key never leaves hardware).
  4. Attach the signature; generate an SBOM (Syft, CycloneDX).
  5. Scan for vulnerabilities with Trivy (fail on CRITICAL).
  6. Upload to S3 (KMS-encrypted, immutable, 5-year retention).
  7. Write a detailed audit trail (job ID, builder identity, commits, timestamp).
  8. Continuous CVE scanning (daily SBOM rescan, auto-notify consumers).

Signing & scanning

sign_firmware.pyCloudHSM signing with provenance metadatapython 
#!/usr/bin/env python3
import subprocess, hashlib, json, os
from datetime import datetime, timezone

def sign_firmware_binary(firmware_path):
    """Sign a firmware binary via CloudHSM."""
    if not os.path.exists(firmware_path):
        raise FileNotFoundError(f"{firmware_path} not found")

    with open(firmware_path, "rb") as f:
        fw_hash = hashlib.sha256(f.read()).hexdigest()
    print(f"[*] Firmware hash: {fw_hash}")

    sign_result = subprocess.run([
        "cloudhsm_client", "sign",
        "--digest", fw_hash,
        "--key-id", os.environ["HSM_KEY_ID"],
        "--algorithm", "SHA256_RSA_PKCS",
    ], capture_output=True, text=True, check=True)

    signature = sign_result.stdout.strip()
    sig_path = f"{firmware_path}.sig"
    with open(sig_path, "w") as f:
        f.write(signature)

    metadata = {
        "firmware": os.path.basename(firmware_path),
        "hash_algo": "SHA256",
        "hash": fw_hash,
        "signed_at": datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ"),
        "signature_algo": "SHA256_RSA_PKCS",
        "hsm_key_id": os.environ["HSM_KEY_ID"],
    }
    with open(f"{firmware_path}.metadata.json", "w") as f:
        json.dump(metadata, f, indent=2)

    print(f"[+] Firmware signed: {sig_path}")
    return sig_path

if __name__ == "__main__":
    import sys
    sign_firmware_binary(sys.argv[1])

Hardware-in-the-loop testing

GPU validation runs on real AMD Instinct silicon in a bare-metal lab: firmware flashing, device enumeration (rocm-smi / amd-smi), memory-bandwidth measurement, thermal monitoring, and benchmarking across the SKU matrix. This is the tier that cannot live on a cloud instance — the hardware must be AMD, which is why these runners are self-hosted and coordinated with IT.

test_harness.pyFlash → enumerate → bandwidth → thermal → JUnitpython 
#!/usr/bin/env python3
import subprocess, re, time

class GPUTestHarness:
    def __init__(self, gpu_model, firmware_path):
        self.gpu = gpu_model
        self.firmware = firmware_path
        self.results = {"gpu": gpu_model, "tests": []}

    def flash_firmware(self):
        print(f"[*] Flashing {self.gpu}...")
        subprocess.run([
            "rocm-flash-firmware", "--device", self.gpu,
            "--firmware", self.firmware, "--verify",
        ], timeout=300, check=True)

    def test_device_enumeration(self):
        output = subprocess.check_output(["rocm-smi"]).decode()
        assert self.gpu in output, f"{self.gpu} not found in rocm-smi output"
        self.results["tests"].append({"name": "device_enumeration", "status": "PASS"})

    def test_memory_access(self):
        output = subprocess.check_output(["rocm-bandwidth-test"]).decode()
        match = re.search(r"Host to Device: ([0-9.]+)", output)
        if match:
            self.results["tests"].append({
                "name": "memory_bandwidth", "status": "PASS",
                "bandwidth_gbps": float(match.group(1)),
            })

    def monitor_thermal(self):
        print("[*] Thermal monitoring (60s)...")
        temps = []
        workload = subprocess.Popen(["rocm-benchmark", "--stress-test", "--duration", "60"])
        start = time.monotonic()
        while time.monotonic() - start < 60:
            temp_out = subprocess.check_output(["rocm-smi", "--showtemp"]).decode()
            match = re.search(r"Temperature.*: ([0-9.]+)", temp_out)
            if match:
                temps.append(float(match.group(1)))
            time.sleep(5)
        workload.wait()
        max_temp = max(temps) if temps else 0
        self.results["thermal"] = {"max_temp_c": max_temp,
                                   "status": "PASS" if max_temp < 85 else "FAIL"}

    def save_junit(self, output_path):
        tests = self.results["tests"]
        xml = '\n'
        xml += f'\n'
        for test in tests:
            xml += f'  \n'
            if test["status"] == "FAIL":
                xml += "    Test failed\n"
            xml += "  \n"
        xml += "\n"
        with open(output_path, "w") as f:
            f.write(xml)

Operational playbook & runbooks

Reliability is owned, not hoped for. These are the scenario runbooks the platform role maintains for the most common failure modes.

Runbook — pre-submit failure

Check the Actions logs first — compile error, build failure, or gate failure? If the compat check fails, verify the firmware/driver version matrix. If the security scan fails, review the findings. If logs are inconclusive, connect via SSM Session Manager (no direct SSH), check disk space and the ccache state, and reproduce locally.

Runbook — cold ccache / slow nightly

If nightly wall-clock spikes, first check ccache hit-rate and the DVC remote. A common cause is a runner replaced without its persistent cache volume re-attached, or a compiler bump invalidating the cache. Re-warm from the shared remote before assuming a real regression.

Runbook — Instinct lab fully booked

Priority queue in the workflows — nightly high, pre-submit lower. Runners check device availability and back off exponentially. Permanent fix: scale the lab with the hardware team or add a device-reservation service.

Runbook — NFS mount fails during build

On the runner, df -h /opt/rocm-tools confirms it's unmounted. Check EFS health, verify the SG allows 2049/tcp from the runner SG, and that mount targets exist in both AZs. Remount, then root-cause (AZ outage? SG change? timeout?) and update the runbook.
