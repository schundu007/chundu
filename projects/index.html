Energy & Oil/Gas Sector

OSDU Data Platform

Architected and deployed the Open Subsurface Data Universe (OSDU) R3 platform on AWS and Azure for major energy enterprises, enabling petabyte-scale seismic and subsurface data management.

Wipro
Feb 2020 – Jun 2023
Remote (Global Clients)

Project Overview

The Open Subsurface Data Universe (OSDU) is an industry-standard data platform that enables oil and gas companies to manage, integrate, and leverage subsurface and wells data at scale. I led the cloud architecture and infrastructure implementation for OSDU R3 across multiple major energy clients, building GPU-accelerated data ingestion and ML training pipelines for petabyte-scale seismic data processing.

This project involved designing multi-tenant landing zones, implementing DevSecOps practices, and ensuring compliance with enterprise security standards across AWS and Azure environments.

Architecture Overview

Multi-tenant OSDU R3 deployment with AKS, Event Hubs, and Cosmos DB

AWS Infrastructure Provisioning

Provisioned OSDU R3 platform on AWS for ExxonMobil and Chevron with enterprise landing zones, GPU-accelerated compute for seismic processing, and petabyte-scale data lake architecture.

  • Landing Zone: Deployed AWS Control Tower with multi-account strategy (workload, shared services, logging, security), SCPs, and AWS Organizations for governance at scale.
  • EKS Clusters: Provisioned production EKS clusters with Karpenter auto-scaling, GPU node groups (p4d/p3 instances) for seismic ML workloads, and Bottlerocket OS for security hardening.
  • Network Architecture: Designed Transit Gateway hub connecting VPCs across regions, AWS PrivateLink for service connectivity, and Direct Connect for on-premises integration.
  • Data Lake: Built petabyte-scale data lake on S3 with Lake Formation for governance, Glue for ETL, and Athena/Redshift Spectrum for analytics on seismic datasets.
  • Streaming Pipeline: Provisioned Amazon MSK (Kafka) and Kinesis for real-time seismic data ingestion at 10+ GB/s throughput with exactly-once delivery guarantees.
  • Security Controls: Implemented AWS IAM Identity Center (SSO), Secrets Manager with automatic rotation, GuardDuty, Security Hub, and CloudTrail for FedRAMP compliance.

Azure Infrastructure Provisioning

Provisioned OSDU R3 platform on Azure for BP, Shell, and TotalEnergies with enterprise-scale landing zones, multi-tier architecture, and Azure-native services for data management and analytics.

  • Landing Zone: Deployed Azure Enterprise-Scale landing zone with Management Groups hierarchy, Azure Policy initiatives, and subscription vending for multi-tenant isolation.
  • AKS Clusters: Provisioned AKS clusters across 3 tiers (Frontend, Platform, Data Services) with availability zones, Azure CNI overlay, and GPU node pools (NC/ND series) for ML training.
  • Network Architecture: Designed hub-spoke topology with Azure Virtual WAN, ExpressRoute for hybrid connectivity, and Azure Firewall Premium for IDPS and TLS inspection.
  • Data Services: Provisioned Cosmos DB (multi-region write), Azure Event Hubs (Kafka protocol), Azure Storage (ADLS Gen2) with hierarchical namespace for seismic data organization.
  • API Layer: Deployed Azure API Management with OAuth 2.0/OIDC, rate limiting, and developer portal for OSDU API consumption by partner applications.
  • CI/CD Automation: Built Azure DevOps pipelines with Terraform modules, environment promotion gates, and integration with Azure Policy for compliance-as-code.
