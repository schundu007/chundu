OSDU Data Platform

Wipro

Remote (Global)

Project Overview

The Open Subsurface Data Universe (OSDU) is an industry-standard data platform that enables oil and gas companies to manage, integrate, and leverage subsurface and wells data at scale. I led the cloud architecture and infrastructure implementation for OSDU R3 across multiple major energy clients.

Built GPU-accelerated data ingestion and ML training pipelines for petabyte-scale seismic data processing, designed multi-tenant landing zones, and implemented DevSecOps practices for enterprise compliance.

Architecture Overview

Multi-tenant OSDU R3 deployment with AKS, Event Hubs, and Cosmos DB

AWS Infrastructure

  • Landing Zone: AWS Control Tower with multi-account strategy, SCPs, and AWS Organizations.
  • EKS Clusters: Production EKS with Karpenter, GPU node groups (p4d/p3), and Bottlerocket OS.
  • Data Lake: Petabyte-scale on S3 with Lake Formation, Glue ETL, and Athena analytics.
  • Streaming: Amazon MSK (Kafka) and Kinesis for 10+ GB/s seismic data ingestion.
  • Security: IAM Identity Center, Secrets Manager, GuardDuty, and Security Hub for FedRAMP.

Azure Infrastructure

  • Landing Zone: Enterprise-Scale with Management Groups and Azure Policy initiatives.
  • AKS Clusters: Multi-tier architecture with GPU node pools (NC/ND series) for ML training.
  • Data Services: Cosmos DB multi-region, Event Hubs (Kafka), and ADLS Gen2.
  • API Layer: Azure API Management with OAuth 2.0 and developer portal.
  • CI/CD: Azure DevOps pipelines with Terraform modules and compliance-as-code.
